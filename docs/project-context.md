# ITMO Labs Web: Project Context

## Что это за проект

Этот репозиторий содержит web-версию лабораторных работ ITMO, перенесенную с legacy Flash/ActionScript реализации на современный стек Vite + TypeScript.

Основная цель текущей реализации: воспроизвести поведение лабораторных стендов, прежде всего Lab 6, в браузере с собственной доменной моделью, конфигурацией оборудования и canvas-рендерингом.

## Технологический стек

- Vite
- TypeScript
- HTML + CSS
- Canvas rendering

Скрипты из `package.json`:

- `npm run dev` - локальная разработка
- `npm run build` - production build через `tsc && vite build`
- `npm run preview` - локальный просмотр production-сборки

## Структура проекта

Ключевые директории в `src/`:

- `src/app/` - прикладной слой и связывание UI/домена
- `src/config/` - конфигурация лабораторий, размеры, порты, ограничения
- `src/domain/` - доменная логика, состояние лаборатории, валидация, физика
- `src/rendering/` - отрисовка, в том числе canvas
- `src/styles/` - стили приложения

Legacy-исходники сохранены отдельно в `sources/itmo-labs/`. Там лежат оригинальные `.as`, `.fla` и `.swf` файлы, которые используются как источник поведения и архитектурных решений при переносе.

## Текущее состояние Lab 6

Ключевая точка доменной логики Lab 6:

- `src/domain/lab6/lab6-laboratory.ts`

Класс `Lab6Laboratory` отвечает за оркестрацию состояния лаборатории. Он управляет:

- текущим этапом лаборатории;
- режимом указателя;
- размещенным оборудованием;
- соединениями между узлами;
- измерениями;
- статусом лаборатории;
- вариантом и положением вентиля;
- служебными идентификаторами и таймингом запуска.

Внутри `Lab6Laboratory` используются:

- `Grid` - работа с тайловой сеткой и размещением;
- `Lab6Physics` - физическая модель/вычисления;
- `Lab6Validator` - проверка корректности схемы и действий.

## Конфигурация и рендеринг Lab 6

Основные точки:

- `src/config/lab6/lab6.config.ts` - размеры оборудования, конфигурация портов, ограничения и параметры Lab 6;
- `src/rendering/canvas/canvas-renderer.ts` - canvas-рендеринг оборудования и схемы.

Это важно учитывать при любых изменениях: геометрия, точки подключения и визуальное представление лаборатории разделены между конфигурацией, доменной моделью и рендерингом.

## Практическое замечание

Если нужно продолжать перенос поведения из Flash, смотреть стоит одновременно в две стороны:

- в `sources/itmo-labs/` как в источник оригинальной логики;
- в `src/domain/lab6/`, `src/config/lab6/` и `src/rendering/` как в текущую web-реализацию.
